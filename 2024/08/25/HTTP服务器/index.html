1.2.3 location模块 – 根据uri匹配配置功能

location中匹配优先级：

1、完全匹配项

2、^~前缀匹配（如果是最长的前缀）

3、按配置文件中出现的顺序，找到第一个正则匹配项

4、前缀匹配（先缓存最长的）

5、/ 万能匹配

使用nginx搭建静态资源服务器，配置文件指定文件根目录：root显式拼接，alias隐式替换

默认显示文件：index file

变量使用

http{
    server{
    	location ~ /([a-z]*)([0-9]*) {	# 下面这个$1为匹配第一个括号的内容，$2为匹配第二个括号的内容
            return 200 "/$1--$2\n";
        }
        location /test/ {
            return 200 '$uri';	# 输出请求的uri
        }
        location /test/ {
            return 200 'arg test';	# 取出test关键字的值?test=value
        }
    }
}

2.3 workflow设计特点

1、所有任务都必须放入序列中进行执行，所有单个任务都默认放进了一个序列

2、序列之间是并行的

3、若某事情需要等待所有序列任务都完成后执行，则使用并行任务

4、并行任务也是任务，又可以放入序列

6.1 路由模式

direct模式（单播）：只要queue的binging key和消息的key完全一致，exchange就可以转发

fanout模式（广播）：交换器不做工作，消息绑定到所有队列中，追求极致性能

topic模式（组播）：通配符匹配key

headers模式（精准，慢）：使用消息的header来确定投发

7 微服务

单体应用耦合性太高，难以开发，性能较差，运维困难，因此需要使用微服务分离业务。

函数调用 ==> 网络通信 ==> HTTP通信慢 ==> 远程过程调用

7.1 REST和RPC区别

REST是具体的接口设计风格，RPC是具体的技术。都是解决将单个应用拆分为微服务

REST基于HTTP，兼容性，可读性强，消耗大；RPC是二进制传输，兼容性差一些，但消耗少，远程调用函数。

7.2 RPC的实现原理 – 代理模式

client ==> client proxy ==> socket <==> socket <== server proxy <== server

客户端存储服务器的代理，然后通过socket通信，服务端认为是内部的代理调用的函数

8.4 Lua语法

1、语法规范

标识符：和C语言相同，下划线，字母，数字，对大小写敏感

注释：–

变量：直接是文件名是全局变量， local标识局部变量。如果一个变量没赋值，则默认为nil空值

赋值：变量是对值的引用，不区分类型，可直接进行引用替换，分配的空间垃圾自动回收

类型：number，string，boolean，table，function，thread，userdata（C数据结构交互）。type(num)类型

2、语法特点

boolean类型中只有false和nil为否，其它都为真

and: left and right，如果left真，返回right，如果left为假，返回left

or: left or right, 如果left真，返回left, 如果left为假，返回right

not: not num, 如果num为真，返回false，如果num为假，返回true

数字全是浮点数，支持+ - * / ^ 加减乘除次方操作，不取整

args.lua文件，参数默认从1开始，0为文件名，-1 为lua，-2及之后为nil空但不报错

执行文件 a b c 为传入参数
